Migrating ATS Scoring to Gemini AI

Replacing manual rule-based heuristics with full semantic analysis using Google Gemini. Simpler code, greater accuracy, and enhanced persistence.

Migration Overview

The Challenge

The previous ATS scoring function was failing. It relied on a fragile mixture of keyword matching algorithms, custom regex normalizations, static scoring structures, and separate concurrent Gemini API calls for work experience and project evaluation. This approach suffered from parse errors, mathematical inconsistencies, and high latency due to multiple LLM roundtrips.

The Solution

We migrated the complete evaluation logic to a single unified prompt executing on Gemini 3.1 Flash-Lite. The new method sends the entire parsed resume structure and the target job description together. Gemini evaluates keywords, skills, experience, education, and projects concurrently under defined guidelines, returning a single schema-conforming JSON payload.

Mongoose Persistence Fix

During the migration, we identified that missingKeywords was declared in the TypeScript interface but absent in the Mongoose schema. This meant missing keywords identified by the engine were never stored in MongoDB. We updated the Mongoose model to persist the array appropriately.
